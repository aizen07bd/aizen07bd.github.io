이제 본격적으로 Rust의 독특한 철학이 드러나는 변수와 불변성을 배워볼 차례입니다. 다른 언어와 가장 차별화되는 지점이라 블로그에 쓸 내용이 정말 풍부할 거예요.

------------------------------
## 1. 기본은 '불변(Immutable)'
Rust에서 변수는 기본적으로 값을 바꿀 수 없습니다.
```rust
fn main() {
    let x = 5;
    println!("x의 값: {}", x);
    x = 6; // ❌ 에러 발생!
}
```

* 왜 이렇게 만들었을까? 값이 변하지 않는다는 확신이 있으면 버그가 줄어들고, 나중에 배울 '동시성(멀티스레딩)' 환경에서 매우 안전하기 때문입니다.
* 에러 메시지의 친절함: 위 코드를 컴파일하면 Rust는 "cannot assign twice to immutable variable"이라며 친절하게 해결책까지 알려줍니다.

## 2. 바꾸고 싶다면 mut 키워드 (Mutable)
변수의 값을 변경해야 할 때는 mut를 붙여서 "이 변수는 바뀔 수 있다"고 명시해 줘야 합니다.
```rust
fn main() {
    let mut x = 5; // mut 추가
    println!("x의 이전 값: {}", x);
    x = 6; // ✅ 정상 작동
    println!("x의 현재 값: {}", x);
}
```

## 3. 변수 vs 상수 (let vs const)
"바뀌지 않는 값"이라면 상수(const)도 있는데, let과 몇 가지 차이가 있습니다. [^1],[^2] 

* const: 절대 변하지 않는 값. 반드시 타입을 명시해야 하며, 프로그램 전체에서 사용 가능합니다. (예: const MAX_POINTS: u32 = 100,000;)
* let: 함수 내에서 사용되며, 기본적으로 불변이지만 mut를 통해 가변성을 선택할 수 있습니다. [^3],[^4] 

## 4. 섀도잉 (Shadowing) - Rust만의 특징
이미 선언된 변수와 같은 이름으로 새 변수를 선언하는 기능입니다.
```rust
fn main() {
    let space = "   "; // 문자열 타입
    let space = space.len(); // 숫자 타입으로 재탄생! (Shadowing)
    println!("공백 길이는: {}", space);
}
```

* 장점: space_str, space_int처럼 이름을 지저분하게 나열할 필요가 없습니다.
* 가변(mut)과의 차이: 섀도잉은 아예 새로운 변수를 만드는 것이라 데이터 타입까지 바꿀 수 있다는 게 핵심입니다.

------------------------------

## 5. 이런것이 궁금해요.
학습과정에서 궁금한걸 미리 물어보고 답을 구했어요. 함께 확인해봐요.

### 1) Shadowing에서 자주 쓰이는 패턴 (함수들)
사실 .len()은 특정 함수라기보다 "데이터의 가공" 절차라고 보시면 됩니다. 섀도잉은 주로 "데이터의 형태를 바꾸거나 정제할 때" 사용해요.

* 타입 변환 (Parsing): 가장 많이 쓰입니다. 외부에서 입력받은 문자열을 숫자로 바꿀 때 유용해요.
```rust
let guess = "42"; 
let guess: u32 = guess.parse().expect("숫자가 아닙니다");
```

* 공백 제거 및 가공 (Trimming):
```rust
let name = "  Rustacean  ";
let name = name.trim(); // "Rustacean"
```

* 중간 계산 결과 저장:
```rust
let price = 100;
let price = price * 1.1; // 세율 적용 후 다시 price라는 이름을 사용
```

------------------------------
### 2) Shadowing vs mut : 왜 둘 다 있을까?
"섀도잉이 되면 그냥 다 섀도잉으로 하면 되지, 왜 mut를 써?"라는 의문이 생길 수 있죠. 하지만 둘의 용도는 완전히 다릅니다.

| 구분 | Shadowing (let) | 가변성 (let mut) |
|---|---|---|
| 핵심 개념 | 새 변수를 같은 이름으로 다시 선언 | 기존 변수의 값을 수정 |
| 타입 변경 | 가능 (문자열 → 숫자 등) | 불가능 (처음 정해진 타입 유지) |
| 사용 사례 | 데이터 형식을 변환하여 깔끔한 이름을 유지할 때 | 반복문 등에서 동일한 데이터를 계속 업데이트할 때 |

예시 비교:

* mut가 필요한 경우: "통장에 돈이 100원 있었는데, 50원이 추가되어 150원이 되었다." (상태의 변화)
* Shadowing이 필요한 경우: "사용자가 입력한 값은 '5'라는 '문자'였는데, 계산을 위해 '5'라는 '숫자'로 바꾸겠다." (데이터의 재정의)

------------------------------
### 3) 헷갈리지 않을까요? (가독성 문제)
네, 너무 남용하면 당연히 헷갈립니다! 그래서 나름의 기준을 가지고 쓰는 게 좋아요.

* 헷갈림 방지 팁: 섀도잉은 보통 코드의 앞부분(준비 단계)에서 데이터를 정제할 때만 짧게 쓰는 것이 관례입니다. 수십 줄 뒤에서 갑자기 섀도잉을 하면 읽는 사람이 당황하겠죠?
* 컴파일러의 도움: Rust 컴파일러는 매우 똑똑해서, 섀도잉된 이후에는 이전 변수를 아예 볼 수 없게 격리해 줍니다. 덕분에 이전 값과 섞여서 발생하는 버그는 오히려 줄어듭니다.
