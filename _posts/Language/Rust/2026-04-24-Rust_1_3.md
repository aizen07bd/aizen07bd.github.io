Rust의 데이터 타입은 크게 두 가지 바구니로 나뉩니다. 하나는 딱 하나의 값만 담는 스칼라(Scalar) 타입이고, 다른 하나는 여러 값을 묶어서 담는 복합(Compound) 타입이에요.

------------------------------
## 1. 스칼라 타입 (단일 값)

### ① 정수형 (Integers)
소수점이 없는 숫자입니다. Rust는 메모리를 얼마나 쓸지 아주 세세하게 결정할 수 있어요.

* 종류: i8, i16, i32, i64, i128, isize (부호 있는 정수) / u8~u128, usize (부호 없는 정수)
* 팁: 보통은 i32를 기본으로 사용합니다. 가장 빠르거든요! usize는 배열의 인덱스나 메모리 주소를 다룰 때 주로 씁니다.

### ② 부동 소수점 (Floating-Point)
소수점이 있는 숫자입니다.

* 종류: f32, f64
* 팁: Rust의 기본 실수 타입은 f64입니다. 현대 CPU에서는 f32와 속도 차이는 거의 없으면서 더 정밀하기 때문이죠.

### ③ 불리언 (Boolean)
참(true) 또는 거짓(false) 딱 두 가지만 가집니다. 논리 구조(if문 등)를 만들 때 필수예요.

### ④ 문자 (Character)
알파벳 하나, 한글 한 글자, 심지어 이모지(😀) 하나도 담을 수 있습니다.

* 특징: 반드시 작은따옴표(')를 사용합니다. (큰따옴표는 문자열!)

------------------------------
## 2. 복합 타입 (여러 값의 묶음)

### ① 튜플 (Tuple)
서로 다른 타입의 값들을 하나로 묶을 때 씁니다. 한 번 선언하면 길이를 늘리거나 줄일 수 없어요.
```rust
let my_tuple: (i32, f64, u8) = (500, 6.4, 1);
let (x, y, z) = my_tuple; // 구조 분해(Destructuring)로 꺼내기
println!("y의 값: {}", my_tuple.1); // 마침표(.)로 접근하기
```

### ② 배열 (Array)
같은 타입의 값들을 여러 개 묶을 때 씁니다. 튜플처럼 길이가 고정되어 있어요.
```rust
let a = [1, 2, 3, 4, 5];
let second = a[1]; // 대괄호([])로 접근하기
```
* 주의: 배열은 데이터가 메모리의 스택(Stack)이라는 공간에 고정된 크기로 들어갑니다. 만약 길이를 마음대로 늘리고 싶다면 나중에 배울 벡터(Vector)를 써야 해요.

------------------------------

## 3. 이런것이 궁금해요.
학습과정에서 궁금한걸 미리 물어보고 답을 구했어요. 함께 확인해봐요.

구조 분해 방식 말고도 마침표(.)와 인덱스를 이용해 직접 꺼내는 방법이 있습니다!
다른 언어의 배열이 my_tuple[0]처럼 대괄호를 쓰는 것과 달리, Rust의 튜플은 마침표를 사용한다는 점이 독특해요.

### 1) 인덱스로 직접 접근하기
튜플의 각 요소에 순번(0부터 시작)을 매겨서 필요한 것만 쏙 골라낼 수 있습니다.
```rust
fn main() {
    let my_tuple = (500, 6.4, "Rust");

    let first = my_tuple.0;  // 500
    let second = my_tuple.1; // 6.4
    let third = my_tuple.2;  // "Rust"

    println!("첫 번째 값: {}", first);
}
```

### 2) 왜 구조 분해(Destructuring)를 더 많이 쓸까?

* 인덱스 접근 (.0, .1): 튜플에서 특정 값 하나만 필요할 때 편리합니다.
* 구조 분해 (let (x, y, z) = ...): 튜플 안에 든 모든 값을 각각의 변수로 담아 한꺼번에 활용해야 할 때 코드가 훨씬 깔끔해집니다.

### 3) (심화) 값이 필요 없을 때의 팁
구조 분해를 할 때 모든 값이 다 필요하지 않다면 언더스코어(_)를 사용해 무시할 수 있습니다.
```rust
let my_tuple = (1, 2, 3);
let (x, _, z) = my_tuple; // 두 번째 값은 무시하고 x와 z만 생성
```

### 4) 왜 .x가 안 되나요?
결론부터 말씀드리면, let A = my_tuple.x 형태로는 사용하지 않습니다.
그 이유는 Rust의 튜플(Tuple)이 각 값에 '이름'이 아닌 '순서(인덱스)'만 부여된 자료형이기 때문이에요.
튜플은 익명(Anonymous)의 데이터 묶음입니다. 
우리가 let (x, y, z) = my_tuple;이라고 썼을 때, x라는 이름은 튜플 안에 원래 있던 이름이 아니라 우리가 데이터를 꺼내면서 새로 붙여준 변수 이름일 뿐입니다.

* my_tuple.0 (O) : "첫 번째 칸에 있는 데이터를 줘"라는 뜻
* my_tuple.x (X) : "x라는 이름표가 붙은 데이터를 줘"라는 뜻 (하지만 튜플엔 이름표가 없어요!)

### 5) 이름으로 접근하고 싶다면? → 구조체(Struct)
만약 질문하신 것처럼 .x나 .name 같이 이름으로 데이터를 꺼내고 싶다면, 튜플 대신 구조체(Struct)를 사용해야 합니다.
```rust
// 구조체 정의
struct Point {
    x: i32,
    y: i32,
}
fn main() {
    let p = Point { x: 10, y: 20 };
    
    // 이제 마침표 뒤에 이름을 쓸 수 있습니다!
    println!("x 좌표: {}", p.x); 
}
```
