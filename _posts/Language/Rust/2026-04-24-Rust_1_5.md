Rust의 제어 흐름(Control Flow)은 프로그램의 실행 방향을 결정하는 아주 중요한 부분입니다. 
특히 Rust의 if와 loop는 다른 언어와 달리 '값'을 반환할 수 있다는 점이 매력적인데요. 

------------------------------
## 1. 조건문: if 표현식
가장 기본이 되는 조건문입니다. [^1] 

* 기본 특징: 조건식에 괄호()를 쓰지 않습니다. 대신 조건은 반드시 불리언(bool) 타입이어야 합니다. (0이나 1을 참/거짓으로 쓰면 에러가 나요!)
* 값을 반환하는 if: 이게 Rust의 핵심입니다! if를 변수에 대입하는 용도로 쓸 수 있어요.
```rust
fn main() {
    let condition = true;
    
    // if가 값을 반환하므로 변수에 바로 대입 가능!
    let number = if condition { 5 } else { 6 }; 

    println!("결과값: {}", number);
}
```
* 주의: if와 else 블록에서 반환하는 데이터의 타입은 반드시 같아야 합니다.

## 2. 무한 루프: loop
조건 없이 영원히 반복합니다. break를 만나야만 멈춰요.

* 값 반환하기: break 뒤에 값을 적어주면 루프를 끝내면서 그 값을 밖으로 던져줍니다. 작업의 성공 여부나 결과값을 전달할 때 유용해요.
```rust
let mut counter = 0;
let result = loop {
    counter += 1;
    if counter == 10 {
        break counter * 2; // 20을 반환하며 루프 종료
    }
};
```

## 3. 조건부 반복: while
조건이 참인 동안 계속 실행합니다. 다른 언어와 거의 비슷합니다. [^2] 


## 4. 컬렉션 순회: for
안전성과 효율성 때문에 Rust에서 가장 권장되는 반복문입니다. 인덱스를 일일이 관리하지 않아도 되어 버그가 적습니다.
```rust
fn main() {
    let a = [10, 20, 30, 40, 50];

    for element in a {
        println!("값: {}", element);
    }
    
    // 일정 범위를 반복하고 싶을 때 (1부터 3까지)
    for number in (1..4) {
        println!("{}!", number);
    }
}
```

## 5. 패턴 매칭 도구: match
Rust의 match는 다른 언어의 switch와 비슷해 보이지만, 훨씬 강력하고 안전한 '패턴 매칭' 도구입니다. Rust 프로그래밍의 꽃이라고도 불리죠.

### 1) 기본 구조
match는 값을 여러 패턴과 비교하여 일치하는 팔(arm)의 코드를 실행합니다.
```rust
let number = 3;
match number {
    1 => println!("일입니다!"),
    2 => println!("이입니다!"),
    3 => println!("삼입니다!"),
    _ => println!("나머지 숫자입니다!"), // 위 패턴에 해당하지 않는 모든 경우
}
```

### 2) match의 핵심 규칙 
Rust의 match가 특별한 이유는 두 가지 규칙 때문입니다.

* 철저함 (Exhaustiveness): 발생 가능한 모든 경우의 수를 다루지 않으면 컴파일 에러가 납니다. 덕분에 "실수로 빼먹은 조건" 때문에 발생하는 버그를 원천 차단합니다.
* 와일드카드 (_): 모든 경우를 일일이 적기 힘들 때, "그 외 나머지 전체"를 의미하는 _ 패턴을 마지막에 반드시 써줘야 합니다.

### 3) 값을 반환하는 match
if와 마찬가지로 match도 표현식이라서 결과를 변수에 바로 담을 수 있습니다.
```rust
let boolean = true;let binary = match boolean {
    true => 1,
    false => 0,
};
```

### 4) 다양한 패턴 활용
단순 숫자뿐만 아니라 범위나 여러 값을 동시에 매칭할 수 있습니다.
```rust
let x = 5;
match x {
    1..=5 => println!("1에서 5 사이군요."), // 범위 매칭
    6 | 7 => println!("6 아니면 7이네요."),   // OR 조건
    _ => println!("그 외의 숫자입니다."),
}
```

## 5. 이런것이 궁금해요.
학습과정에서 궁금한걸 미리 물어보고 답을 구했어요. 함께 확인해봐요.

### 1) 일반적인 사용 방식 (값 반환 없이 실행만)
당연히 일반 프로그래밍에서 쓰던 방식 그대로 사용 가능합니다! 우리가 흔히 아는 "조건에 따라 코드를 실행만 하고 끝내는" 방식도 당연히 기본입니다.
다른 언어처럼 단순히 조건에 맞춰 특정 로직만 수행하고 싶을 때는 이렇게 씁니다.
```rust
fn main() {
    let number = 7;

    if number < 5 {
        println!("5보다 작아요.");
    } else if number == 7 {
        println!("럭키 세븐!");
    } else {
        println!("5보다 크고 7은 아니네요.");
    }
}
```
이때는 변수에 담을 필요가 없으므로 블록({ }) 안에 세미콜론을 찍든 안 찍든 상관없고, 평소 스타일대로 코드를 짜시면 됩니다.

### 2) 다른 언어와 다른 '딱 한 가지' 주의점
방식은 같지만 문법 규칙이 하나 엄격합니다.

* 반드시 bool 타입이어야 함:
* C나 JavaScript에서는 if (1) 또는 if (변수) 처럼 숫자가 있거나 객체가 있으면 참(True)으로 간주하죠?
   * Rust는 안 됩니다. 무조건 결과가 true 아니면 false여야만 컴파일이 됩니다.
   * if number { ... } (❌ 에러) -> if number != 0 { ... } (✅ 정상)

### 3) Rust의 범위(Range) 표현식 두 가지 방식
* 끝 숫자를 포함하지 않는 범위: ..
가장 기본이 되는 형태입니다. 시작..끝이라고 쓰면 '끝' 직전까지만 반복합니다. 수학적으로는 시작 <= x < 끝 형태입니다.
```rust
for number in 1..4 {
    println!("{}", number);
}// 출력 결과:// 1// 2// 3// (4는 포함되지 않습니다!)
```
왜 이렇게 쓰나요? 보통 배열의 길이가 n일 때 인덱스는 0부터 n-1까지이기 때문에, 0..n이라고 쓰면 인덱스 범위와 딱 맞아서 사고를 방지하기 좋습니다.

* 끝 숫자를 포함하는 범위: ..=
끝 숫자까지 포함하고 싶을 때는 등호(=)를 붙여줍니다. 시작..=끝은 수학적으로 시작 <= x <= 끝입니다.
```rust
for number in 1..=4 {
    println!("{}", number);
}// 출력 결과:// 1// 2// 3// 4// (4가 포함됩니다!)
```

### 4) 숫자를 거꾸로 세고 싶을 때: .rev()
범위 뒤에 .rev() 함수를 붙이면 숫자를 역순으로 출력합니다.
```rust
for number in (1..4).rev() {
    println!("{}", number);
}// 출력 결과:// 3// 2// 1
```

### 5) 요약

| 코드 | 의미 | 출력 숫자 |
|---|---|---|
| 1..4 | 1 이상 4 미만 | 1, 2, 3 |
| 1..=4 | 1 이상 4 이하 | 1, 2, 3, 4 |
