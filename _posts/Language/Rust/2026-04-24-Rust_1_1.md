Rust 학습의 첫 단추인 환경 설정 단계입니다. Rust는 rustup이라는 도구 덕분에 설치와 업데이트가 매우 간편한 편이에요. 운영체제별 설치 방법과 핵심 도구인 Cargo 사용법을 정리해 드릴게요.

------------------------------
## 1. Rust 설치하기 (rustup) [^1] 
Rust 공식 설치 관리자인 rustup을 사용하면 컴파일러(rustc)와 패키지 매니저(cargo)가 한 번에 설치됩니다. [^2] 

* Windows:
1. [Rust 공식 홈페이지](https://www.rust-lang.org/tools/install)에서 rustup-init.exe를 다운로드하여 실행합니다.
2. 설치 중 "Visual Studio C++ Build Tools"가 필요하다는 메시지가 뜨면 지시에 따라 설치해야 합니다. (C++ 컴파일러가 Rust의 빌드 과정에서 필요하기 때문이에요.)
3. 설치 옵션은 기본값인 1을 선택하면 됩니다. [^3] 

* Mac / Linux:
1. 터미널을 열고 다음 명령어를 입력합니다:
```sh   
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```   
2. 설치가 완료되면 source $HOME/.cargo/env를 입력하거나 터미널을 재시작합니다.
3. zsh 사용자라면:
```sh
# 1. .zshrc 파일 끝에 Rust 바이너리 경로 추가
echo 'export PATH="$HOME/.cargo/bin:$PATH"' >> ~/.zshrc
# 2. 변경된 설정을 현재 터미널에 즉시 적용
source ~/.zshrc
```
✅ 설치 확인: 터미널(CMD)에서 rustc --version을 입력했을 때 버전 정보가 나오면 성공입니다. [^4] 

------------------------------
## 2. 에디터 설정 (VS Code 추천)
현재 Rust 개발에서 가장 표준적인 조합은 VS Code + rust-analyzer입니다.

1. VS Code를 설치합니다.
2. 확장(Extensions) 마켓플레이스에서 rust-analyzer를 검색해 설치하세요.
* 주의: 이름이 유사한 'Rust'라는 확장이 있지만, 현재는 rust-analyzer가 공식적이며 성능이 훨씬 뛰어납니다. [^5] 
3. 자동 완성, 타입 힌트, 실시간 에러 체크 기능을 제공하므로 학습 효율이 엄청나게 올라갑니다.

------------------------------
## 3. Rust의 핵심 도구, Cargo 익히기
Rust 개발은 시작부터 끝까지 Cargo(카고)와 함께합니다. 빌드, 패키지 관리, 문서 생성 등을 모두 담당하는 만능 도구예요. [^6] 

* 새 프로젝트 만들기:
```sh
cargo new hello_rust
cd hello_rust
```
* 이 명령어를 치면 hello_rust 폴더 안에 src/main.rs 파일과 Cargo.toml 설정 파일이 자동으로 생깁니다.
* 새 프로젝트를 만들면 자동으로 git이 활성화 됩니다. git을 사용하지 않을 경우 cargo new 프로젝트명 --vcs none 를 사용합니다.
* 프로젝트 구조:
    * src/main.rs: 실제 코드가 들어가는 곳입니다.
    * Cargo.toml: 프로젝트의 이름, 버전, 그리고 외부 라이브러리(Dependencies)를 명시하는 설정 파일입니다. (Node.js의 package.json과 비슷해요.) [^7] 
* 자주 쓰는 명령어 3개:
    1. cargo check: 코드가 컴파일되는지 빠르게 확인합니다. (실제 실행 파일을 만들지 않아 가장 빠릅니다. 개발 중 자주 사용해요.)
    2. cargo build: 프로젝트를 빌드하여 target/debug 폴더에 실행 파일을 만듭니다.
    3. cargo run: 빌드와 실행을 한 번에 합니다. [^8]

## 4. Hello world 코드 분석
cargo new로 프로젝트를 만들면 src/main.rs 파일에 자동으로 생성되는 가장 기본적인 코드입니다. 아주 짧지만 Rust의 핵심 규칙들이 담겨 있어요.
```rust
fn main() {
    println!("Hello, world!");
}
```
### 1) fn main() : 프로그램의 시작점

* fn: 'function'의 약자로, 함수를 선언할 때 사용하는 키워드입니다.
* main: Rust 프로그램이 실행될 때 가장 먼저 찾는 이름입니다. 모든 실행 가능한 Rust 프로그램에는 반드시 main 함수가 있어야 해요.
* (): 매개변수(입력값)가 없다는 뜻입니다. 만약 전달할 값이 있다면 이 괄호 안에 적어줍니다.

### 2) { ... } : 함수의 본문

* 함수가 호출되었을 때 실행할 코드들을 중괄호({ }) 안에 작성합니다.

### 3) println!("Hello, world!"); : 화면 출력

* println!: 화면에 텍스트를 출력하고 줄바꿈을 하라는 명령어입니다.
* 중요! 끝에 붙은 느낌표(!)는 이 기능이 일반 함수가 아니라 매크로(Macro)임을 뜻합니다. Rust에서 메타 프로그래밍을 할 때 쓰이는데, 입문 단계에서는 "특수 기능을 하는 함수" 정도로 이해하셔도 충분해요.
* "Hello, world!": 출력할 문자열입니다. Rust는 반드시 큰따옴표를 사용합니다.
* ; (세미콜론): 이 줄(구문)이 끝났음을 알립니다. Rust는 세미콜론 유무에 따라 의미가 달라지는 경우가 많으니 꼭 붙여주세요.

------------------------------

## 5. 이런것이 궁금해요.
학습과정에서 궁금한걸 미리 물어보고 답을 구했어요. 함께 확인해봐요.

### 1) 왜 src 하위에 있어야 하나요?
Cargo는 기본적으로 "소스 코드는 src 폴더에 있다"고 약속되어 있습니다(Convention).

* 이 약속 덕분에 우리가 별도의 설정 없이도 cargo build만 치면 Cargo가 알아서 코드를 찾아 컴파일할 수 있는 거예요.
* 만약 위치를 바꾸고 싶다면 Cargo.toml 파일을 복잡하게 수정해야 하는데, 굳이 권장되지는 않습니다.

### 2) 왜 main.rs여야 하나요? (확장자 주의: .rs입니다!)

* 실행 파일(Binary)을 만들 때: Cargo는 src/main.rs를 찾아서 "아, 이게 프로그램의 시작점이구나!"라고 인식합니다.
* 라이브러리를 만들 때: 만약 실행 목적이 아니라 다른 곳에서 가져다 쓸 도구(Library)를 만든다면, 시작 파일 이름은 src/lib.rs가 되어야 합니다.

### 3) 파일 이름을 다르게 하고 싶다면? (Binaries)
하나의 프로젝트에서 여러 개의 실행 파일을 만들고 싶을 때는 src/bin/ 폴더를 활용합니다.

* src/bin/hello.rs
* src/bin/goodbye.rs
* 이렇게 파일을 만들면 cargo run --bin hello 같은 방식으로 각각 실행할 수 있습니다.

### 4) 확장자는 .rs!
Rust의 공식 확장자는 .rs입니다. (Rust의 줄임말이죠!)
