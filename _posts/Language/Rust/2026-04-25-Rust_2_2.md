소유권의 두 번째 관문인 이동(Move)과 복사(Copy)입니다. 이 개념을 이해하고 나면, "왜 내 코드가 컴파일되지 않는가?"에 대한 의문이 대부분 풀릴 거예요.

------------------------------
## 1. 이동 (Move)
다른 언어(JS, Python 등)에서는 변수 s1을 s2에 대입하면 두 변수가 같은 데이터를 가리키게 되죠? 하지만 Rust의 힙(Heap) 데이터를 담는 객체(예: String)는 다릅니다.
```rust
let s1 = String::from("hello");
let s2 = s1; // 여기서 '이동(Move)'이 발생!
// println!("{}", s1); // ❌ 에러! s1은 더 이상 유효하지 않음
println!("{}", s2);    // ✅ 정상
```

* 왜 이동하나요? 소유권 원칙 2번(값은 한 번에 한 소유자만 가질 수 있다) 때문입니다.
* 만약 s1과 s2가 동시에 소유권을 가지면, 스코프가 끝날 때 메모리를 두 번 해제(Double Free)하려다 프로그램이 터질 수 있거든요. 그래서 Rust는 s2에게 소유권을 넘겨주고 s1은 무효화해 버립니다.

------------------------------
## 2. 복사 (Copy)
그런데 신기하게도 정수나 불리언 같은 타입은 Move가 일어나지 않습니다.
```rust
let x = 5;let y = x; // '복사(Copy)'가 발생

println!("x: {}, y: {}", x, y); // ✅ 둘 다 사용 가능!
```

* 왜 이건 되나요? 정수처럼 스택(Stack)에만 저장되는 고정 크기 타입들은 복사하는 비용이 매우 저렴합니다. 굳이 소유권을 넘기고 자시고 할 것 없이 그냥 값을 똑같이 하나 더 찍어내도 부담이 없기 때문입니다.
* 이렇게 자동으로 복사되는 타입들을 Copy 트레이트(Trait)를 가졌다고 부릅니다. (i32, f64, bool, char 등)

------------------------------
## 3. 클론 (Clone)
만약 힙 데이터(String 등)를 이동시키지 않고 진짜로 복제해서 둘 다 쓰고 싶다면 어떻게 할까요? 그때는 .clone()을 명시적으로 써야 합니다.
```rust
let s1 = String::from("hello");
let s2 = s1.clone(); // 힙 메모리까지 통째로 복사

println!("s1: {}, s2: {}", s1, s2); // ✅ 둘 다 사용 가능!
```

* 주의: clone은 힙 메모리를 새로 할당하므로 데이터가 클수록 성능에 부담을 줄 수 있습니다.

------------------------------
## 📝 팁

* 비교 이미지:
   * Move: 책을 친구에게 빌려주는 게 아니라 '아예 주는 것' (나는 이제 책이 없음)
   * Copy: 종이에 적힌 숫자를 친구가 자기 수첩에 '베껴 적는 것' (나도 숫자 알고 있음)
   * Clone: 책을 한 권 더 인쇄해서 주는 것 (둘 다 책이 있지만 종이값이 듦)

------------------------------
## 💡 다음 단계 예고
이제 소유권을 옮기는 법(Move)과 복제하는 법(Clone)을 배웠습니다. 하지만 매번 소유권을 주고받는 건 너무 번거롭죠.
그래서 등장하는 것이 참조와 빌림(Borrowing)입니다! 소유권은 내가 그대로 갖고, 다른 사람에게 잠시 구경만 시켜주는 방법이에요.
참조자(&)를 이용해 데이터를 빌려오는 방법을 바로 알아볼까요?
