Rust 학습의 가장 큰 고비이자 하이라이트인 2단계: 소유권(Ownership) 커리큘럼입니다. 이 단계는 단순히 문법을 외우는 게 아니라, Rust가 메모리를 관리하는 '철학'을 이해하는 과정이에요.

------------------------------
## 🦀 2단계: 소유권(Ownership) 상세 로드맵

## 1. 소유권의 세 가지 원칙 (3주차 초반)
Rust가 메모리를 관리하는 근본적인 규칙을 배웁니다.

* 학습 내용: 소유권의 3규칙, 변수 스코프(Scope)와 메모리 해제(drop), 스택(Stack) vs 힙(Heap) 메모리 구조.
* 포스팅 주제: "가비지 컬렉터 없이 메모리를 관리한다고? 소유권 3법칙", "메모리의 두 방, 스택과 힙 이해하기".

## 2. 이동(Move)과 복사(Copy) (3주차 중반)
값이 다른 변수로 넘어갈 때 어떤 일이 벌어지는지 다룹니다.

* 학습 내용: Shallow Copy(얕은 복사)와 Deep Copy(깊은 복사)의 차이, Move 현상(소유권 이전), Copy 트레이트(기본 타입들의 동작).
* 포스팅 주제: "내 변수가 사라졌어요! Rust의 Move 현상 파헤치기", "왜 정수는 Move 대신 Copy가 될까?".

## 3. 참조와 빌림 (Borrowing) (4주차 초반)
소유권을 넘기지 않고 '빌려만' 오는 방법을 배웁니다. 실무에서 가장 많이 쓰는 개념입니다.

* 학습 내용: 참조자(&), 가변 참조자(&mut), 빌림의 규칙(공유 참조는 여러 개 가능, 가변 참조는 오직 하나).
* 포스팅 주제: "빌려 쓰기의 미학: & 참조자 활용법", "데이터 경쟁(Data Race)을 막는 가변 참조자의 제약".

## 4. 슬라이스 (Slice) (4주차 후반)
컬렉션의 일부분을 안전하게 참조하는 방법을 배웁니다.

* 학습 내용: 문자열 슬라이스(&str), 배열 슬라이스, 슬라이스를 이용한 안전한 메모리 접근.
* 포스팅 주제: "문자열의 일부분만 안전하게 가져오는 법: Slice", "왜 &String보다 &str을 선호할까?".

------------------------------
## 💡 2단계 학습을 위한 마음가짐

* 컴파일러와 싸우지 마세요: 2단계에서는 컴파일 에러를 정말 많이 마주하게 될 거예요. 하지만 그건 컴파일러가 "이대로 실행하면 나중에 메모리 버그가 터질 거야!"라고 미리 알려주는 친절한 경고입니다.
* 그림을 그려보세요: 메모리 상에서 데이터가 어디로 이동하는지 손으로 그려가며 공부하면 훨씬 빨리 이해됩니다. 블로그 포스팅에도 직접 그린 도식화 그림을 넣으면 인기가 정말 많을 거예요.
