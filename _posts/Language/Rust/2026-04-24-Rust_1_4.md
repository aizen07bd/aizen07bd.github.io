Rust의 함수는 다른 언어와 비슷해 보이지만, '표현식(Expression)'을 활용한 독특한 반환 방식이 매력적입니다. 

------------------------------
## 1. 함수의 기본 선언: fn
함수는 fn 키워드로 시작하며, 이름은 관례적으로 스네이크 케이스(snake_case)를 사용합니다.
```rust
fn main() {
    println!("메인 함수 시작!");
    another_function(5, 'h'); // 함수 호출
}
fn another_function(x: i32, unit_label: char) {
    println!("측정값: {}{}", x, unit_label);
}
```
* 매개변수 타입은 필수: 함수를 정의할 때는 반드시 각 매개변수의 타입을 명시해야 합니다. 컴파일러가 함수가 어떻게 쓰일지 확실히 알게 하기 위해서예요.

## 2. 값을 반환하는 함수
값을 돌려주는 함수는 화살표(->) 뒤에 반환 타입을 적어줍니다.
```rust
fn five() -> i32 {
    5 // 👈 여기에 주목하세요!
}
fn main() {
    let x = five();
    println!("x의 값: {}", x);
}
```

## 3. Rust 함수의 하이라이트: 구문(Statement) vs 표현식(Expression)
이 부분이 블로그 포스팅에서 가장 중요한 핵심 포인트입니다.

* 구문 (Statement): 어떤 동작을 수행하지만 값을 반환하지 않습니다. 끝에 세미콜론(;)이 붙습니다.
* 예: let x = 6;
* 표현식 (Expression): 결과값을 산출합니다. 끝에 세미콜론을 붙이지 않습니다.
* 예: 위 예제의 5는 그 자체로 결과값을 나타내는 표현식입니다.

💡 왜 중요할까?
Rust에서는 함수의 마지막 줄에 세미콜론을 생략하면, 그 줄의 결과값이 자동으로 반환(Return)됩니다. return 키워드를 써도 되지만, 마지막 줄에서는 생략하는 것이 Rust다운(Idiomatic) 방식입니다.

## 4. return 키워드는 언제 쓰나요?
보통 함수 중간에 조건에 따라 일찍 종료하고 값을 반환해야 할 때 명시적으로 사용합니다.
```rust
fn early_return(x: i32) -> i32 {
    if x > 10 {
        return 10; // 함수 중간에 나갈 때는 return 사용
    }
    x + 1 // 마지막 줄은 세미콜론 없이 표현식으로 반환
}
```

## 5. 이런것이 궁금해요.
학습과정에서 궁금한걸 미리 물어보고 답을 구했어요. 함께 확인해봐요.

### 1) 튜플을 이용한 다중 반환
있습니다! Rust에서는 튜플(Tuple)을 활용해서 여러 개의 값을 아주 깔끔하게 반환할 수 있어요.
함수의 반환 타입 부분(->)에 튜플 형식을 적어주고, 마지막 줄(표현식)에서도 튜플을 던져주면 됩니다.
```rust
fn main() {
    let (width, height) = get_dimensions(); // 구조 분해로 바로 받기
    println!("가로: {}, 세로: {}", width, height);
}
fn get_dimensions() -> (i32, i32) {
    let w = 100;
    let h = 200;
    
    (w, h) // 세미콜론 없이 튜플 반환
}
```

### 2) 왜 이렇게 쓰나요?
다른 언어(예: C, Java)에서는 여러 값을 반환하려면 별도의 클래스나 구조체를 정의하거나, 포인터를 써야 해서 번거로운 경우가 많습니다.
반면 Rust는 튜플 덕분에:

* 임시적인 묶음: 굳이 이름을 붙일 필요 없는 가벼운 데이터 묶음을 즉석에서 반환하기 좋습니다.
* 가독성: let (a, b) = 함수(); 처럼 받는 쪽에서도 직관적으로 변수에 담을 수 있습니다.

### 3) 더 복잡한 데이터라면?
만약 반환하는 값이 3~4개를 넘어가고 각각의 의미가 중요하다면, 앞서 잠깐 언급했던 구조체(Struct)를 반환하는 것이 훨씬 좋습니다. 튜플은 .0, .1로 접근해야 해서 나중에 코드를 읽을 때 "이게 뭐였지?" 싶을 수 있거든요.
