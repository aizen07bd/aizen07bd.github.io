2단계의 첫 단추이자, Rust를 지탱하는 가장 중요한 기둥인 소유권(Ownership)과 메모리 구조입니다. 이 내용을 이해하면 "왜 Rust가 안전한가?"에 대한 답을 얻으실 수 있어요.

------------------------------
## 1. 전제 지식: 스택(Stack) vs 힙(Heap)
소유권을 이해하려면 먼저 데이터가 메모리 어디에 사는지 알아야 합니다.

* 스택 (Stack):
   * 특징: 정해진 크기의 데이터만 담습니다. (예: i32, bool, 배열)
   * 장점: 매우 빠릅니다. 함수가 끝나면 알아서 정리됩니다.
* 힙 (Heap):
   * 특징: 크기가 변할 수 있는 데이터를 담습니다. (예: String, Vector)
   * 단점: 스택보다 느립니다. 대신 데이터가 유연하게 늘어날 수 있죠.
   * 연결: 힙에 데이터를 넣으면, 그 데이터가 어디 있는지 알려주는 '주소(포인터)'가 생깁니다. 이 주소값은 스택에 저장됩니다.

------------------------------
## 2. 소유권의 3가지 원칙
Rust는 이 원칙들을 통해 가비지 컬렉터 없이도 메모리를 완벽하게 관리합니다.

   1. Rust의 모든 값은 '소유자(owner)'라고 불리는 변수를 가집니다.
   2. 값은 한 번에 한 소유자만 가질 수 있습니다. (중요: 독점 소유)
   3. 소유자가 스코프(Scope) 밖으로 벗어나면, 값은 버려집니다(drop).

------------------------------
## 3. 메모리 해제의 마법: 스코프(Scope)
다른 언어는 "이 메모리 이제 안 써요!"라고 말해주거나(C), 청소기(GC)가 돌 때까지 기다려야 하지만, Rust는 중괄호{}가 끝나면 즉시 메모리를 반납합니다.
```rust
{                      // s는 아직 유효하지 않음
    let s = String::from("hello"); // s는 이때부터 유효함 (힙 메모리 할당)
    // s를 가지고 작업 수행
}                      // 이 스코프가 끝남! s는 더 이상 유효하지 않음. 
                       // Rust가 자동으로 'drop' 함수를 호출해 메모리 반납.
```

------------------------------
## 4. 왜 이런 규칙이 있을까?

* 메모리 누수 방지: 쓰지 않는 메모리가 계속 쌓이는 걸 막아줍니다.
* 이중 해제(Double Free) 방지: 똑같은 메모리를 두 번 지우려고 해서 발생하는 치명적인 버그를 원천 차단합니다. "한 번에 소유자는 한 명"이라는 규칙 덕분이죠.

------------------------------
여기까지가 소유권의 기본 배경입니다. 이해가 좀 되시나요?
다음 단계는 이 규칙 때문에 발생하는 아주 독특한 현상인 '이동(Move)'에 대해 알아볼 차례입니다. "내 변수가 왜 다른 변수에 값을 주니까 사라졌지?" 하는 상황을 경험하게 되실 거예요. 

## 5. 이런것이 궁금해요.
학습과정에서 궁금한걸 미리 물어보고 답을 구했어요. 함께 확인해봐요.

### 1) C (수동 관리)
C와 GC는 메모리를 관리하는 서로 다른 두 가지 대표적인 방식을 의미합니다. Rust가 왜 대단한지 비교할 때 항상 등장하는 개념이에요.
* 의미: C언어 같은 오래된 프로그래밍 언어의 방식을 말합니다.
* 특징: 개발자가 직접 메모리를 빌려오고(malloc), 다 썼으면 직접 반납(free)해야 합니다.
* 단점:
   * 반납을 까먹으면 메모리가 꽉 차버립니다 (메모리 누수).
   * 이미 반납한 걸 또 반납하면 프로그램이 터집니다 (이중 해제).
   * 즉, 사람이 실수할 확률이 매우 높습니다.

### 2) GC (Garbage Collector, 쓰레기 수집기)

* 의미: Java, Python, JavaScript, Go 같은 언어들이 사용하는 방식입니다.
* 특징: 프로그램 실행 중에 '청소기(GC)'가 배경에서 돌아다니며, "이제 이 메모리 안 쓰는 거 같네?" 하고 알아서 치워줍니다.
* 단점:
   * 청소기가 돌 때 프로그램이 아주 잠깐씩 멈출 수 있습니다 (성능 저하).
   * 메모리를 언제 정확히 치울지 예측하기 어렵습니다.

### 3) 그렇다면 Rust는? (스코프의 의미)
Rust는 위 두 방식의 장점만 합쳤습니다.

* C처럼 청소기(GC)가 없어서 속도가 매우 빠릅니다.
* GC처럼 개발자가 직접 메모리를 치우라고 코드를 짤 필요가 없습니다.

그 비결이 바로 '스코프(Scope)'입니다.
컴파일러가 코드를 훑으면서 "아, 이 변수는 이 중괄호(})가 끝나면 더 이상 안 쓰겠구나!"라고 미리 계산해서, 그 자리에 메모리 반납 코드를 몰래 삽입해 버립니다.

* C: 사람이 직접 치워야 함 (위험함)
* GC: 나중에 청소기가 치워줌 (느려짐)
* Rust: 스코프가 끝나면 자동으로 치워지게 설계됨 (빠르고 안전함)

------------------------------
