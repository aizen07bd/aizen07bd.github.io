드디어 3단계의 첫 문을 여시네요! Rust의 구조체(Struct)는 여러 데이터를 하나로 묶어 의미 있는 '물건'을 만드는 설계도와 같습니다.

------------------------------
## 1. 일반 구조체 (Classic Struct)
가장 많이 쓰이는 방식입니다. 각 데이터에 이름(Field Name)을 붙여서 관리합니다.

* 정의: struct 키워드 뒤에 이름을 쓰고, 중괄호 안에 이름: 타입을 적습니다.
* 사용: 인스턴스를 만들 때 필드 이름을 명시해야 해서 가독성이 매우 좋습니다.
```rust
// 1. 정의
struct User {
    username: String,
    email: String,
    active: bool,
}
fn main() {
    // 2. 사용 (인스턴스 생성)
    let mut user1 = User {
        email: String::from("someone@example.com"),
        username: String::from("rust_user"),
        active: true,
    };

    // 3. 값 읽기와 수정 (마침표 사용)
    user1.email = String::from("another@example.com");
    println!("사용자 이름: {}", user1.username);
}
```

------------------------------
## 2. 튜플 구조체 (Tuple Struct)
구조체에 이름은 있지만, 내부 필드에는 이름이 없고 타입만 있는 형태입니다. 튜플처럼 순서대로 접근합니다.

* 언제 쓰나요? 구조체 전체에 이름을 붙여서 다른 튜플과 구분하고 싶지만, 필드 하나하나에 이름을 붙이는 게 오히려 거추장스러울 때 씁니다. (예: 좌표, 색상)
```rust
struct Color(i32, i32, i32);
struct Point(i32, i32, i32);
fn main() {
    let black = Color(0, 0, 0);
    let origin = Point(0, 0, 0);
    
    // black과 origin은 똑같이 (0,0,0)을 담지만 타입이 서로 달라 안전합니다!
    println!("Red 값: {}", black.0);
}
```

------------------------------
## 3. 유닛 구조체 (Unit-Like Struct)
필드가 아예 없는 구조체입니다.

* 언제 쓰나요? 데이터는 필요 없지만, 어떤 상태나 동작을 정의하고 싶을 때 씁니다. 나중에 배울 '트레이트(Trait)'를 구현할 때 주로 등장합니다.
```rust
struct AlwaysAlive;
fn main() {
    let _subject = AlwaysAlive;
}
```

------------------------------
## 📝 블로그 꿀팁: "필드 초기화 축약법"
변수 이름과 구조체 필드 이름이 같다면 줄여서 쓸 수 있어요.
```rust
fn build_user(email: String, username: String) -> User {
    User {
        email,    // email: email 대신 축약 가능!
        username, // username: username 대신 축약 가능!
        active: true,
    }
}
```

## 💡 정리 및 다음 단계

   1. 일반 구조체: 데이터에 이름이 필요할 때 (가장 권장)
   2. 튜플 구조체: 데이터 이름보다 타입 구분이 중요할 때
   3. 유닛 구조체: 데이터 없이 '타입' 자체만 필요할 때

구조체의 생김새가 이제 눈에 들어오시나요? 다음 단계로는 이 구조체에 '함수'를 달아서 생동감을 불어넣는 메소드(Method)와 impl 블록에 대해 알아볼께요.
