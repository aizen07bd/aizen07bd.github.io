소유권이 '물건을 아예 넘겨주는 것'이라면, 참조와 빌림(Borrowing)은 물건을 주인은 그대로 둔 채 '빌려만 주는 것'입니다. 실무에서 소유권 이동(Move)보다 훨씬 더 많이 쓰이는 아주 중요한 개념이에요.

------------------------------
## 1. 참조자 (&)와 빌림
함수에 값을 넘길 때 소유권을 넘기지 않고 참조자(&)를 전달하면, 함수가 끝나도 원래 변수를 계속 사용할 수 있습니다. 이를 '빌림'이라고 합니다.
```rust
fn main() {
    let s1 = String::from("hello");

    // &를 붙여서 소유권이 아닌 '참조'를 전달 (빌려줌)
    let len = calculate_length(&s1);

    println!("'{}'의 길이는 {}입니다.", s1, len); // ✅ s1을 여전히 쓸 수 있음!
}
fn calculate_length(s: &String) -> usize { // 타입을 &String으로 받음
    s.len()
} // 여기서 s가 스코프를 벗어나도, 소유권이 없으므로 실제 메모리는 'drop'되지 않음
```

------------------------------
## 2. 가변 참조자 (&mut)
기본적으로 빌려온 값은 수정할 수 없습니다(읽기 전용). 만약 빌려온 값을 수정하게 하려면 가변 참조자를 써야 합니다.
```rust
fn main() {
    let mut s = String::from("hello");

    change(&mut s); // 가변으로 빌려줌
    println!("{}", s); // "hello, world" 출력
}
fn change(some_string: &mut String) {
    some_string.push_str(", world"); // ✅ 빌려온 값 수정 가능
}
```

------------------------------
## 3. 빌림의 두 가지 황금 규칙 (매우 중요!)
Rust는 메모리 안전을 위해 빌릴 때 딱 두 가지 규칙을 엄격히 지키게 합니다. 블로그 포스팅의 핵심 내용이 될 거예요.

   1. 공유 참조(&T)는 동시에 여러 개 가능: "여러 명이 동시에 책을 읽는 건 괜찮아."
   2. 가변 참조(&mut T)는 딱 하나만 가능: "누군가 책에 글을 쓰고 있다면, 다른 사람은 읽지도 쓰지도 못해."
   * 이유: 여러 곳에서 동시에 값을 바꾸려고 하면 데이터가 꼬이는 '데이터 경합(Data Race)'이 발생하기 때문입니다.
   
------------------------------
## 4. 댕글링 참조 (Dangling References) 방지
Rust는 빌려준 주인이 먼저 사라지는 상황을 컴파일 단계에서 차단합니다.
```rust
fn main() {
    let reference_to_nothing = dangle();
}
fn dangle() -> &String { // ❌ 에러 발생!
    let s = String::from("hello");
    &s // s는 함수가 끝나면 사라지는데, 그 주소를 밖으로 빌려주려고 함
}
```

------------------------------
## 📝 팁

* 비유:
   * 공유 참조: 도서관에서 여러 명이 같은 책을 읽고 있는 상태.
   * 가변 참조: 편집자가 원고를 수정하는 중. 이때는 다른 사람이 읽으면 수정 중인 불완전한 내용을 볼 수 있으니 접근을 막음.
* 결론: "Rust는 빌림 규칙을 통해 런타임 에러가 아닌 컴파일 타임에 메모리 버그를 잡아낸다"는 점을 강조하세요.
