2단계 소유권의 마지막 주제인 슬라이스(Slice)입니다. 슬라이스는 데이터의 전체가 아닌 '연속된 일부분'을 가리키는 특수한 종류의 참조자예요. [^1] 
소유권을 갖지 않으면서 특정 범위만 안전하게 관리할 수 있어 실무에서 굉장히 자주 쓰입니다.

------------------------------
## 1. 문자열 슬라이스 (&str)
가장 대표적인 슬라이스입니다. 문자열의 일부분을 [시작_인덱스..끝_인덱스] 형식으로 지정합니다.
```rust
fn main() {
    let s = String::from("hello world");

    let hello = &s[0..5];  // "hello" (0부터 4까지)
    let world = &s[6..11]; // "world" (6부터 10까지)

    println!("{} {}", hello, world);
}
```

* 주의: 앞서 배운 .. 범위 표현식과 같습니다. 0..5는 0, 1, 2, 3, 4번 인덱스를 포함합니다.
* 생략 문법:
   * &s[..5] : 처음부터 5까지
   * &s[6..] : 6부터 끝까지
   * &s[..] : 전체

## 2. 왜 슬라이스를 쓰나요? (안전성)
만약 슬라이스가 없다면, 우리는 문자열의 특정 범위를 나타내기 위해 "시작 인덱스"와 "길이"라는 숫자를 따로 관리해야 합니다.
그런데 만약 데이터 원본이 삭제되거나 수정된다면 어떻게 될까요? 그 숫자들은 쓸모없어지고 버그의 원인이 됩니다. 하지만 슬라이스는 '빌림(Borrowing)'이기 때문에, 원본이 유효한지 컴파일러가 계속 체크해 줍니다.

## 3. 함수의 매개변수로 쓰는 슬라이스
블로그에 꿀팁으로 적기 좋은 내용입니다. 함수를 만들 때 &String보다 &str을 매개변수 타입으로 쓰는 것이 훨씬 유연합니다.
```rust
fn main() {
    let my_string = String::from("hello world");

    // &String(전체)도, &str(일부분)도 모두 받아들일 수 있음!
    first_word(&my_string[..]); 
    first_word("hello world"); // 문자열 리터럴도 사실은 슬라이스(&str)
}
fn first_word(s: &str) -> &str {
    // 공백을 찾아 첫 단어 슬라이스를 반환하는 로직...
    &s[..5]
}
```

## 4. 배열 슬라이스
문자열뿐만 아니라 일반 배열도 슬라이스를 만들 수 있습니다. [^2] 
```rust
let a = [1, 2, 3, 4, 5];let slice = &a[1..3]; // [2, 3]을 가리키는 슬라이스
```

## 📝 팁

* 핵심 정리:
   1. 슬라이스는 소유권이 없는 참조자의 일종이다.
   2. 데이터의 범위를 숫자로 따로 관리할 때 생기는 위험을 방지해 준다.
   3. &str은 String보다 더 범용적인 타입이다. [3] 
* 시각화: 긴 띠(String)에서 특정 영역만 점선으로 표시하고 그곳을 가리키는 화살표를 그려보세요.

## 5. 이런것이 궁금해요.
학습과정에서 궁금한걸 미리 물어보고 답을 구했어요. 함께 확인해봐요.

슬라이스에서도 ..= 형식을 당연히 사용하실 수 있습니다!
앞서 반복문(for)에서 배우셨던 범위 표현식이 슬라이스 인덱스 지정에도 그대로 적용된다고 보시면 돼요.

### 1) 끝 숫자를 포함하는 슬라이스 (..=)
..=를 사용하면 지정한 마지막 인덱스의 값까지 포함해서 가져옵니다.
```rust
fn main() {
    let s = String::from("hello");

    // 인덱스 0, 1, 2를 포함 (h, e, l)
    let slice1 = &s[0..2];  
    
    // 인덱스 0, 1, 2를 포함 (h, e, l)
    let slice2 = &s[0..=2]; 

    println!("slice1 (..): {}", slice1); // "he" 출력
    println!("slice2 (..=): {}", slice2); // "hel" 출력
}
```

### 2) 왜 ..를 더 자주 쓸까요?
실무에서는 ..=보다 ..를 훨씬 더 많이 사용하는데, 여기에는 몇 가지 이유가 있습니다.

* 수학적 편리함: [0..5]라고 하면 5 - 0 = 5라는 계산을 통해 슬라이스의 길이를 바로 알 수 있습니다.
* 인덱스 안전성: 배열의 길이가 5일 때, 마지막 인덱스는 4입니다. 이때 전체를 가져오려면 [0..5]라고 쓰면 되는데, 만약 ..=를 쓰려면 [0..=4]라고 써야 해서 숫자를 하나 빼는 계산을 한 번 더 해야 하죠.

### 3) 정리

| 표현식 | 의미 | 결과 (데이터가 "abcde"일 때) |
|---|---|---|
| &s[0..2] | 0부터 2 미만 (0, 1) | "ab" |
| &s[0..=2] | 0부터 2 이하 (0, 1, 2) | "abc" |

------------------------------
### 4)💡 2단계 마무리 퀴즈
소유권과 슬라이스를 잘 이해하셨는지 확인해 볼 수 있는 아주 유명한 케이스가 있습니다. 아래 코드가 왜 에러가 날지 한 번 생각해 보세요! 
```rust
fn main() {
    let mut s = String::from("hello world");
    let word = &s[0..5]; // s를 '불변'으로 빌림

    s.clear(); // ❌ 여기서 에러 발생! (s를 '가변'으로 수정 시도)

    println!("첫 단어: {}", word); // 빌려온 참조자를 나중에 사용함
}
```

이유: word라는 슬라이스가 s의 일부분을 빌리고(Borrow) 있는데, 원본인 s를 싹 비워버리면(clear) word가 가리키는 곳이 미궁에 빠지기 때문입니다. Rust는 이런 상황을 절대 용납하지 않아요!

이 부분이 Rust의 '대여 규칙(Borrowing Rules)'에서 가장 핵심적인 부분이에요.
결론부터 말씀드리면, mut로 선언된 변수라도 상황에 따라 &(불변)로 빌려줄 수도 있고, &mut(가변)로 빌려줄 수도 있습니다.
### 5) 왜 둘 다 가능한가요?
변수를 let mut s로 선언했다는 건, "이 주인(s)은 값을 바꿀 권한이 있다"는 뜻입니다. 주인이 권한을 많이 가지고 있으니, 빌려줄 때도 두 가지 옵션이 생기는 거죠.

* &s (불변 빌림): "내가 바꿀 순 있지만, 너한테는 읽기 권한만 줄게. 건드리지는 마."
* &mut s (가변 빌림): "내가 바꿀 권한이 있으니, 너한테도 수정할 권한까지 줄게."

반대로 let s(불변)로 선언했다면, 주인도 바꿀 권한이 없으니 &mut s로 빌려주는 것 자체가 불가능합니다.

### 6) Rust의 '빌림 규칙'
Rust 컴파일러는 빌려줄 때 아래 규칙을 철저히 감시합니다.

   1. 불변 참조(&s)는 동시에 여러 명에게 빌려줄 수 있다.
   * 데이터를 읽기만 하는 건 여러 명이어도 안전하니까요.
   2. 가변 참조(&mut s)는 딱 한 명에게만 빌려줄 수 있다.
   * 누군가 고치고 있는데 다른 사람이 읽거나 또 고치면 데이터가 꼬이기 때문입니다.
   3. 가변으로 빌려줬다면, 그동안은 불변으로도 못 빌려준다.
   * 누가 고치는 중인데 옆에서 읽고 있으면, 읽는 도중에 값이 변해버릴 수 있으니까요.
   
### 7) 앞선 예제 상황 분석
```rust
let mut s = String::from("hello");let word = &s[0..5]; // ✅ '불변'으로 빌려줌 (읽기만 해!)

s.clear(); // ❌ 에러! (가변 권한을 사용해 값을 고치려 함)
println!("{}", word); // 빌려준 상태가 유지되어야 하는데 주인이 값을 바꿔버리면 안 됨!
```

이 예제에서 word는 읽기 전용 슬라이스(&str)가 필요했기 때문에 &s 형태로 빌려온 것입니다. 하지만 "빌려준 동안에는 주인도 값을 바꿀 수 없다"는 규칙 때문에 s.clear()에서 에러가 난 것이죠.


## 📝 블로그 포스팅 요약

* 제목: "가변 변수(mut)라고 항상 가변으로만 빌려주나요? No!"
